02

Functional Requirements Table

User stories expressed as structured functional requirements with actor, priority, and acceptance criteria.

ID Actor User Story Acceptance Criteria Priority
FR-01 Student As a student, I want to log in with my university credentials so I can access my personalized dashboard. Student logs in via university ID + password. Session is created. Redirect to dashboard. High
FR-02 Student As a student, I want to see only the courses relevant to my major and academic year. Dashboard shows courses filtered by major and academicYear matching the logged-in student. High
FR-03 Student As a student, I want to view all approved materials for a course I'm enrolled in. Clicking a course shows a list of materials with approvalStatus = APPROVED only. High
FR-04 Student As a student, I want to upload a material for one of my courses so it can be reviewed. Upload form accepts title, description, and file. Material is saved with approvalStatus = PENDING. Confirmation shown. High
FR-05 Moderator As a moderator, I want to see all pending materials so I can review them. Moderator dashboard shows all materials with PENDING status. High
FR-06 Moderator As a moderator, I want to approve a pending material so it becomes visible to students. Clicking Approve sets approvalStatus = APPROVED. Material appears in student course view. High
FR-07 Moderator As a moderator, I want to reject a material with a reason so the uploader understands why. Clicking Reject prompts for a rejectionReason. Status becomes REJECTED. High
FR-08 Moderator As a moderator, I want to delete any material that violates content policies. Delete button removes the material from DB and cloud storage. Confirmation dialog shown first. High
FR-09 Moderator As a moderator, I want to upload a material that is automatically approved. When a moderator uploads, approvalStatus is automatically set to APPROVED. Medium
FR-10 Mod / Admin As an admin, I want to assign the moderator role to a student. Admin can set roles = [STUDENT, MODERATOR] for any Student. (Future) Low (Future)
FR-11 Admin As an admin, I want to add new courses and manage majors/faculties. Admin CRUD panel for Course, Major, Faculty models. (Future) Low (Future)
FR-12 Student As a student, I want to download approved materials from a course. Each material row shows a download/view link pointing to fileUrl. Medium
